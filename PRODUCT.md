# Product

## Register

product

## Users

Busy professionals drowning in email. They open NeatMail during work hours, often between meetings or at the start and end of the day. Their primary job: triage incoming messages, know what needs attention, and clear the rest without manual sorting or context-switching.

## Product Purpose

NeatMail uses AI to organize Gmail and Outlook inboxes in real time. It categorizes emails into actionable labels, auto-archives noise, generates context-aware draft replies, and alerts users where they prefer (Telegram). The outcome is fewer hours spent on email and more mental bandwidth for real work.

## Brand Personality

Clean, sharp, focused. The interface should feel like a well-organized desk: nothing unnecessary, everything where you expect it. Calm and competence, not flash or friction. The AI works in the background; the user stays in control.

## Anti-references

- Over-designed SaaS: gradient-heavy hero sections, metric-stat grids, animated icon illustrations, SaaS template clichés
- Traditional enterprise email clients: dense data tables, cluttered sidebars, information overload, Gmail's busy default UI
- "Delight" through decoration: confetti, bounce animations, excessive branding, personality that competes with utility

## Design Principles

1. **Do more with less.** Every pixel, word, and interaction earns its place. Remove before adding. Whitespace is a feature, not waste.
2. **Calm through control.** The interface reduces cognitive load by showing only what matters now. Order breeds calm.
3. **Context-aware, not context-stealing.** AI labels, drafts, and notifications respect attention. Background work surfaces only when useful.
4. **Expert confidence.** Trust the user. Don't over-explain or patronize. Power users should feel the tool scales with them, not talks down to them.
5. **Clarity over cleverness.** Every element should be instantly understandable. No mystery icons, no ambiguous labels, no hidden affordances.

## Accessibility & Inclusion

Target WCAG 2.1 AA compliance:
- Color contrast ≥4.5:1 for body text, ≥3:1 for large text
- Keyboard-navigable throughout, visible focus indicators
- Screen-reader labels on all interactive elements
- Respect `prefers-reduced-motion` for all animations
- Digest palette high-contrast mode: sufficient contrast on critical/attention status colors
